Superficie A — Chat no nosso app BYOK

Painel de chat dentro do Mara Sandra Connect. Cada usuario cola a chave do provider dele (Claude/OpenAI). Multiprovider via adapter. Auth = sessao Supabase.

Superficie B — Claude / ChatGPT do usuario

Nosso sistema vira ferramenta conectada: servidor MCP para o Claude e connector (OpenAPI) para o ChatGPT. Auth por token pessoal (PAT) que preserva o RLS.

Mind map — experiencia nos clientes externos (Claude / ChatGPT)

Interno — dentro do Claude/ChatGPT dele

Conectar (uma vez)
  • Em Configuracoes do app, gera um Personal Access Token (mostrado 1x).
  • No Claude: adiciona o connector MCP (URL + token). No ChatGPT: ativa a Action.
  • O token mapeia para o usuario interno → acesso total via RLS.
O que ele pede em linguagem natural "Crie um caso pra cliente nova Maria Silva, CPF X, aposentadoria por idade."
"Atualize o caso #123 pra fase judicial."
"Abra uma solicitacao de CNIS no caso da Joana."
criar/editar cliente criar/atualizar caso andamento + comentario criar e responder solicitacao anexar documento
Guarda-corpos
  • Escrita = preview + confirmar (host + nossa tool).
  • Nenhuma tool de deletar existe.
  • Tudo auditado em ia_acoes.

Parceiro — dentro do Claude/ChatGPT dele

Conectar (uma vez)
  • Gera o PAT dele na tela de Configuracoes.
  • Conecta no Claude/ChatGPT igual ao interno.
  • Token mapeia para o parceiro → RLS limita aos casos dele.
O que ele pede "Comente no caso do meu cliente que protocolei o documento."
"Anexe este RG ao caso do Pedro."
"Marque a solicitacao de CNIS como atendida."
criar cliente (ao abrir caso) criar/atualizar caso dele andamento + comentario responder solicitacao anexar documento editar dados do cliente criar solicitacao qualquer caso de outro parceiro
Por que alguns ficam riscados Reflete o RLS real de producao: clientes_update e solicitacoes_modify sao is_interno() apenas. A IA nem mostra essas ferramentas ao parceiro.
